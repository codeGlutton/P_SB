**데이터 테이블(csv 파일) 생성 규칙**

1. 데이터 기록이 시작되는 첫 행은 Range 와 그 값(Data, Struct, All 중 하나)으로 기록되야한다.
-> Data 타입의 경우 속한 폴더명의 구조체 타입을 틀로 두고 테이블 값만 추출
-> Struct 타입의 경우 속한 폴더명의 구조체 타입을 부모로 두고 추가적인 Field 값들로 새로운 틀 생성 (유일하게 부모 구조체로 사용가능. 자식들은 "파일명+Children 폴더" 아래에 저장할 것)
-> All 타입의 경우 속한 폴더명의 구조체 타입을 부모로 두고 추가적인 Field 값들로 새로운 틀 생성 및 데이터를 추출

ex1) 스폰이 가능한 테이블들을 모아 두고 싶고, 그 구조가 모두 같을 경우 == Struct 타입으로 스폰가능 틀을 생성하고 Children 폴더 안에 Data 타입들로 player, athlete 등의 테이블을 채워야 함
ex2) 스폰이 가능한 테이블들을 모아 두고 싶고, 그 구조가 모두 다를 경우 == 공통된 부분만 Struct 타입으로 스폰가능 틀을 생성하고 Children 폴더 안에 All 타입들로 player, athlete 등의 테이블을 채워야 함
ex3) 스폰이 가능한 테이블들을 모아 두고 싶고, 그 구조가 일부 다를 경우 == 공통된 부분만 Struct 타입으로 스폰가능 틀을 생성하고 Children 폴더 안에 All, Data 타입들로 player, athlete 등의 테이블을 채워야 함


2. Range 행 이후, 등장하는 두번째 행은 열의 타입형을 기록한 행이다. 그리고 세번째 행은 열 별칭을 기록한 행이다.
-> 상속에 관계없이 모두 필수적인 열이 존재한다. 0번째 열은 Id, 1번째 열은 ServerName, 2번째 열은 ClientName이다.
-> 기본 타입형의 경우, 첫 글자만 대문자로 나머지는 소문자로 작성해야한다.
-> 언리얼 혹은 c++ 자료구조 등의 기타 타입 형식은 기존처럼 대소문자를 구분해주어야 한다.
-> <> 내부 언리얼 인터페이스 타입은 지원하지 않지만, 예외적으로 TSoftClassPtr은 지원한다.
-> "Server"가 별칭에 prefix로 붙으면 서버측만 보이며, "Client"가 별칭에 prefix로 붙은면 클라측에만 보인다. 아무것도 붙지 않는 경우 모두 보인다.

※ 상속관계에 따라 부모의 구조와 동일해야 한다
 

3. 이후 등장하는 행은 모두 데이터이다.
-> TSoftClassPtr는 에셋 주소를 따음표없이 작성할 것
-> 문자열, 숫자 등의 기본 유형은 모두 그냥 기록할 것

※ Struct 타입의 경우 없어도 무방하다
